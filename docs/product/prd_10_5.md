# PRD 10.5 — Peer/Comparable Analysis

**Status**: Draft
**Feature**: #8 from Bloomberg/YCharts parity roadmap
**Depends on**: Existing company detail + ML scoring endpoints

## Problem

Users cannot compare companies side by side. To evaluate whether AAPL or MSFT is the better buy, they must open two browser tabs and mentally cross-reference metrics. Every professional terminal (Bloomberg, YCharts, Morningstar) offers a peer comparison view.

## Solution

A new `/compare` page allowing side-by-side comparison of 2-5 companies across ~25 financial metrics. No new backend endpoints — the page fetches data from existing per-ticker APIs in parallel.

## User Flow

1. Navigate to `/compare` (via NavBar → Signals → Compare, or from a stock detail page)
2. Search for companies by name or ticker in the search bar
3. Selected companies appear as removable chips; comparison table populates progressively
4. Metrics are grouped by category (Scores, Valuation, Profitability, Growth, etc.)
5. Best/worst values per row are highlighted green/red for quick scanning
6. URL updates with selected tickers (`?tickers=AAPL,GOOGL,MSFT`) — shareable
7. Export comparison as CSV

## Data Sources

Per ticker, two parallel fetches:
- `GET /v1/company/{ticker}/summary?include_evidence=false` — deterministic scores, fundamental ratios, market metrics, rank
- `GET /v1/predictions/score/{ticker}` — ML probability, key ratios, classification, market cap

Both are optional — companies may have one, both, or neither scoring system. Null values display as em-dash.

## Metric Groups

| Group | Metrics |
|-------|---------|
| Overview | Country, Sector, Market Cap |
| Deterministic Scores | Overall, Fundamental, Market, Rank |
| ML Signals | Probability, Weight, Classification, Composite Score |
| Valuation | P/E, P/B |
| Profitability | ROE, Net Margin, Gross Margin, Operating Margin |
| Growth | Revenue Growth, EPS Growth |
| Risk & Capital | Debt/Equity, Current Ratio, FCF Yield |
| Performance | 1-Year Return, Max Drawdown |

Metric values prefer ML `key_ratios` where available, falling back to deterministic `fundamental_ratios`.

## UI Design

- **Layout**: Transposed table — companies as columns, metrics as rows
- **Ticker selector**: Search input with autocomplete dropdown, chips for selected tickers, max 5
- **Highlighting**: Best value per row gets subtle green background, worst gets red (3+ companies only)
- **Sticky column**: Metric label column stays visible on horizontal scroll (mobile)
- **Progressive loading**: Columns appear as individual API responses arrive
- **Empty states**: "Add at least 2 companies to compare" when < 2 selected; rows with all-null values hidden

## Entry Points

1. NavBar → Signals → "Compare"
2. Stock detail page header → "Compare" button (pre-populates current ticker)
3. Direct URL: `/compare?tickers=AAPL,GOOGL`

## Acceptance Criteria

1. Navigate to `/compare` — shows search bar and empty state
2. Search and add 2-3 companies — table populates with all metric groups
3. Metrics formatted correctly (percentages, multiples, currency)
4. Best/worst values highlighted per row
5. Remove a ticker via chip × — column disappears, URL updates
6. CSV export downloads comparison data
7. From stock detail, "Compare" button navigates with ticker pre-populated
8. URL is shareable — `/compare?tickers=AAPL,GOOGL` loads both companies
9. Handles missing data gracefully (em-dash for nulls, hidden all-null rows)
10. `npm run build` succeeds
