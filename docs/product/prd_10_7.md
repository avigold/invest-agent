# PRD 10.7 — Multi-Ticker Chart Overlay

**Status**: Draft
**Feature**: #9 from Bloomberg/YCharts parity roadmap
**Depends on**: Existing chart endpoint + Compare page (PRD 10.5)

## Problem

Users comparing companies on the Compare page see a metrics table but cannot visualise relative price performance. To compare how AAPL and MSFT performed over the past year, they must open separate stock detail pages and mentally cross-reference two charts.

## Solution

A new `CompareChart` component embedded in the Compare page. Renders all selected tickers (2–5) as normalised percentage-return lines on a single lightweight-charts instance. No new backend endpoints — data fetched in parallel from the existing per-ticker chart API.

## User Flow

1. Navigate to `/compare` and add 2+ tickers
2. Chart appears above the comparison table showing all tickers as percentage lines
3. Each ticker has a distinct colour; legend shows ticker + current value
4. Period selector (1M, 3M, 6M, 1Y, 5Y) controls time range
5. Crosshair tracks all lines simultaneously
6. Adding/removing tickers updates the chart in real time

## Data Source

Existing endpoint, called once per ticker:
```
GET /v1/company/{ticker}/chart?period={period}
```

Returns `{ points: [{date, value, volume}] }`. Client-side normalisation converts absolute prices to percentage returns from the latest common start date.

## Normalisation

All tickers shown as percentage change from a common base:
1. Find the latest start date across all tickers' data
2. For each ticker, find the first price on or after that date (`base`)
3. Each point: `pct = ((value - base) / base) * 100`

This matches the existing benchmark overlay pattern in StockChart.

## UI Design

- **Chart**: Full-width, dark theme (matching existing StockChart)
- **Series**: `LineSeries` per ticker, 2px line width, no area fill
- **Colours**: Blue (#3b82f6), Amber (#f59e0b), Emerald (#10b981), Violet (#8b5cf6), Red (#ef4444)
- **Legend**: Coloured dot + ticker text, updates on crosshair move to show current % value
- **Period selector**: Button group below chart (1M, 3M, 6M, 1Y, 5Y)
- **Y-axis format**: `+12.3%` / `-5.1%`
- **Empty state**: Shows only when < 2 tickers selected (chart not rendered)
