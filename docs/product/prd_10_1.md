# PRD 10.1 — CSV Export

**Status**: Complete
## Problem Statement

Bloomberg and YCharts let users download any data table as CSV/XLSX. Invest Agent has 6 data-heavy table pages (Countries, Industries, Companies, Fundamentals, ML Picks, Watchlist) but no export capability. Users cannot take data into their own spreadsheets for further analysis.

## Solution

Add a pure frontend CSV export utility. All data is already loaded in the browser — no backend changes needed. Each table page gets an "Export CSV" button that downloads the currently visible/filtered data.

## Implementation

**New file**: `web/src/lib/export.ts` — shared `exportToCsv(filename, headers, rows)` utility using Blob + `<a download>`. RFC 4180 compliant escaping. No new dependencies.

## Acceptance Criteria

1. `npm run build` succeeds
2. Export button visible on all 6 table pages
3. Clicking export downloads a `.csv` file named `{table}_{YYYY-MM-DD}.csv`
4. CSV opens correctly in Excel/Numbers with no mangled columns
5. Applied filters (country, sector, search) are reflected in the exported data
