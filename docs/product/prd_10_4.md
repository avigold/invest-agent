# PRD 10.4 — Live Screener + Saved Configs (Feature Roadmap #6)

**Status**: In Progress

## Problem

The current screener is a historical pattern analysis tool — it runs a background job that finds stocks achieving X% returns in Y-year windows, then analyses what features those winners shared. This is powerful for research but slow (minutes per run) and cannot answer the most common screening question: "Show me all stocks where P/E < 20 and ROE > 15%."

Bloomberg EQS, YCharts, and Finviz all offer real-time metric-based screening with saved configurations and pre-built templates. Our screener lacks this fundamental capability. Users also cannot save or reuse screening configurations (Feature Roadmap #6).

## Solution

Add a **live filter-based screener** that filters the current universe (~136 companies) on financial metrics instantly, alongside saved screener configurations and pre-built templates. The existing historical pattern screener is preserved under a separate tab.

### Filterable Metrics (24 fields)

| Category | Fields |
|----------|--------|
| **Descriptive** | Country, Sector, ML Classification, Deterministic Classification |
| **Valuation** | P/E Ratio, P/B Ratio |
| **Profitability** | ROE, ROA, Net Margin, Gross Margin, Operating Margin |
| **Growth** | Revenue Growth, EPS Growth |
| **Financial Health** | Debt/Equity, Current Ratio, Interest Coverage |
| **Income** | Dividend Yield, FCF Yield |
| **ML Signals** | ML Probability, Fundamental Score, Market Score, Company Score |
| **Performance** | Momentum 12M, Max Drawdown 12M |

### Filter Operators

- Numeric: `>`, `<`, `≥`, `≤`, `=`, `between`
- Categorical: `equals`, `in` (multi-select)

### Pre-built Templates (6)

| Template | Filters |
|----------|---------|
| Value | P/E < 15, P/B < 2, Debt/Equity < 2 |
| Growth | Revenue Growth > 15%, EPS Growth > 15%, Momentum 12M > 10% |
| Dividend | Dividend Yield > 2%, FCF Yield > 5%, Debt/Equity < 3 |
| Quality | ROE > 15%, Net Margin > 10%, Current Ratio > 1.5, Debt/Equity < 1.5 |
| Momentum | Momentum 12M > 20%, Max DD 12M > -20% |
| ML High Conviction | ML Probability > 0.7 |

## Data Source

All metrics come from `PredictionScore.feature_values` (186 features from the ML pipeline). P/E and P/B are derived from latest price + EPS/book value (same logic as PRD 10.3). Deterministic scores computed via `score_from_features()`. Universe is ~136 companies — all filtering is synchronous in-memory.

## API

### Live Screen Endpoints

```
GET  /v1/screener/live/fields    — Field metadata for filter builder UI
POST /v1/screener/live/screen    — Apply filters, return matching companies
POST /v1/screener/live/export    — CSV download of filtered results
```

**Screen request:**
```json
{
  "filters": {
    "rules": [
      {"field": "sector", "op": "in", "value": ["Information Technology"]},
      {"field": "pe_ratio", "op": "lt", "value": 20},
      {"field": "roe", "op": "gt", "value": 0.15}
    ]
  },
  "sort_by": "probability",
  "sort_desc": true,
  "limit": 100,
  "offset": 0
}
```

**Screen response:**
```json
{
  "total": 42,
  "items": [
    {
      "ticker": "AAPL",
      "company_name": "Apple Inc.",
      "country": "US",
      "sector": "Information Technology",
      "pe_ratio": 32.5,
      "pb_ratio": 48.2,
      "roe": 0.15,
      "net_margin": 0.25,
      "probability": 0.72,
      "det_classification": "Buy",
      "...": "..."
    }
  ]
}
```

### Saved Screen Endpoints

```
GET    /v1/screener/saved          — List user's saved screens + templates
POST   /v1/screener/saved          — Save a new screen config
PUT    /v1/screener/saved/{id}     — Update a saved screen config
DELETE /v1/screener/saved/{id}     — Delete a saved screen config
```

## Database Change

New table `saved_screens`:

| Column | Type | Notes |
|--------|------|-------|
| id | UUID | Primary key |
| user_id | UUID | FK → users (CASCADE) |
| name | String(200) | Screen name |
| description | Text | Optional description |
| filters | JSONB | Filter rules |
| sort_by | String(50) | Sort column |
| sort_desc | Boolean | Sort direction (default true) |
| columns | JSONB | Optional display column list |
| created_at | DateTime | |
| updated_at | DateTime | |

Index on `user_id`. Migration: `0016_add_saved_screens.py`.

Templates are stored as Python constants, not DB rows.

## UI

### Screener Page Tabs

The `/screener` page gets two tabs:
- **"Screen"** (default) — new live filter-based screener
- **"Historical Patterns"** — existing job-based screener (unchanged)

### Filter Builder

- Top bar: template/saved-screen selector, "Save" button, "Clear All", "Export CSV"
- Vertical list of filter rules, each with: field dropdown (grouped by category) → operator dropdown → value input → remove button
- "+ Add Filter" button
- Results update on filter change (debounced 300ms)

### Results Table

- Default columns: Ticker, Name, Country, Sector, P/E, ROE, Revenue Growth, ML Prob, Classification
- Sortable column headers
- Ticker links to `/stocks/{ticker}`
- Match count header: "42 companies match"
- Empty state when no matches
