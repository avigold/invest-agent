# PRD 10.8 — Relative Valuation (Company vs Sector Medians)

**Status**: Draft
**Feature**: #10 from Bloomberg/YCharts parity roadmap
**Depends on**: ML scoring pipeline (PredictionScore), Company module, Industry module

## Problem

Users see a company's key ratios (P/E, ROE, margins) on the Stock Detail page but cannot judge whether those values are good or bad relative to sector peers. A P/E of 25 is meaningless without knowing the sector median is 18 (expensive) or 35 (cheap). Professional terminals show percentile rankings — we have nothing comparable.

Additionally, different sectors use fundamentally different valuation multiples. Financials and Real Estate should never show P/E (distorted by provisions and FFO-based valuation respectively). Tech emphasises growth and margins. Users need sector-appropriate metrics, not a one-size-fits-all ratio card.

## Solution

A new **"Valuation vs Peers"** section on the Stock Detail page, placed between Key Ratios and ML Score. For each sector-relevant metric, a horizontal **dot-on-range chart** shows where the company sits within its GICS sector's distribution.

**Backend**: Precomputed sector percentile statistics stored in a new `sector_valuation_stats` table, recomputed during `score_sync`. A new API endpoint returns the company's ratios alongside sector stats.

**Frontend**: A self-fetching `ValuationVsPeers` component with CSS-based dot-on-range bars, metric-aware colour coding, and sector-specific metric selection.

## User Flow

1. Navigate to any Stock Detail page (e.g. `/stock/AAPL`)
2. Below Key Ratios, see "Valuation vs Information Technology Peers (28 companies)"
3. Each metric shows a horizontal range bar with the company's position as a coloured dot
4. Green dot = favourable (high ROE, low P/E), red = unfavourable, yellow = near median
5. For Financials/Real Estate, P/E is intentionally excluded with sector-appropriate metrics shown instead

## Data Source

Uses existing `PredictionScore.feature_values` (186 features from ML system) for fundamentals. P/E and P/B computed from latest price + feature_values (same logic as `routes_predictions.py` lines 494–506). Peer grouping is **global** (all companies in same GICS sector regardless of country) since per-country gives too few peers.

## Sector-Specific Multiples

Each GICS sector gets a curated set of 5–6 metrics:

| GICS | Sector | Multiples | Rationale |
|------|--------|-----------|-----------|
| 10 | Energy | P/E, P/B, FCF Yield, Debt/Equity, ROE, Dividend Yield | Capital-intensive, FCF matters |
| 15 | Materials | P/E, P/B, Gross Margin, ROE, Debt/Equity, FCF Yield | Cyclical, asset-heavy |
| 20 | Industrials | P/E, P/B, Operating Margin, ROE, Revenue Growth, Debt/Equity | Mixed capital models |
| 25 | Consumer Disc. | P/E, Revenue Growth, Gross Margin, ROE, FCF Yield, P/B | Growth + profitability |
| 30 | Consumer Staples | P/E, Dividend Yield, Gross Margin, ROE, Debt/Equity, P/B | Stable, dividend focus |
| 35 | Health Care | P/E, Revenue Growth, Gross Margin, ROE, FCF Yield, P/B | Pipeline/growth driven |
| 40 | Financials | P/B, ROE, Net Margin, Debt/Equity, Dividend Yield | **NO P/E** — distorted by provisions |
| 45 | Info Tech | P/E, Revenue Growth, Gross Margin, Operating Margin, ROE, P/B | Growth + margins |
| 50 | Comm Services | P/E, Revenue Growth, EBITDA Margin, ROE, FCF Yield, P/B | Platform economics |
| 55 | Utilities | P/E, Dividend Yield, Debt/Equity, ROE, FCF Yield, P/B | Regulated, income-focused |
| 60 | Real Estate | P/B, Dividend Yield, Debt/Equity, ROE, FCF Yield | **NO P/E** — FFO-based valuation |

## Data Model

### New table: `sector_valuation_stats`

| Column | Type | Description |
|--------|------|-------------|
| id | UUID | PK |
| gics_code | String(2) | GICS sector code ("10", "45", etc.) |
| sector_name | String(100) | Display name ("Energy", "Information Technology") |
| as_of | Date | Computation date |
| calc_version | String(50) | Version for methodology tracking |
| company_count | Integer | Number of companies in the sector |
| metrics | JSONB | Percentile breakpoints per metric |
| created_at | DateTime(tz) | Row creation timestamp |

Unique constraint on `(gics_code, as_of, calc_version)`.

`metrics` JSONB structure:
```json
{
  "pe_ratio": {"p10": 8.2, "p25": 12.1, "p50": 18.3, "p75": 25.4, "p90": 38.1},
  "roe":      {"p10": -0.05, "p25": 0.04, "p50": 0.12, "p75": 0.20, "p90": 0.28}
}
```

## API

### `GET /v1/company/{ticker}/peer-valuation`

Returns the company's ratios alongside precomputed sector percentile stats.

**Response**:
```json
{
  "ticker": "AAPL",
  "sector": "Information Technology",
  "gics_code": "45",
  "company_count": 28,
  "metrics": [
    {
      "key": "pe_ratio",
      "label": "P/E Ratio",
      "format": "multiple",
      "higher_is_better": false,
      "company_value": 32.5,
      "percentile_rank": 72,
      "stats": { "p10": 8.2, "p25": 15.3, "p50": 22.1, "p75": 30.8, "p90": 45.2 }
    }
  ]
}
```

`percentile_rank` is interpolated at request time from the five precomputed breakpoints. `company_value` is null when the ratio cannot be computed (e.g. negative earnings for P/E).

## Precomputation

Runs in `score_sync_handler` after signal change detection (line 174):

1. Load all `PredictionScore` rows for the active model
2. Load `CompanyPriceHistory` for scored companies
3. Group by GICS sector (map `PredictionScore.sector` → GICS code)
4. For each sector, compute p10/p25/p50/p75/p90 for each sector-relevant metric
5. Outlier handling: exclude P/E < 0 or > 200
6. Upsert `SectorValuationStats` rows

## UI Design

### Dot-on-Range Chart

Each metric renders as a horizontal row:

```
P/E Ratio      ──────|──[  ●──]────────  32.5x  72nd %ile
Revenue Growth ──[──●─]────────────────  +12.3%  45th %ile
Gross Margin   ─────────[──●─]─────────  42.1%   80th %ile
```

- Thin line: p10 to p90 range (whiskers)
- Thicker box: p25 to p75 (interquartile range)
- Vertical tick: p50 (median)
- Coloured dot: company's position
  - Green: favourable (≥60th effective percentile)
  - Yellow: near median (40th–60th)
  - Red: unfavourable (<40th)
- "Effective" percentile inverts for "lower is better" metrics (P/E, P/B, Debt/Equity)

### Header

"Valuation vs {Sector Name} Peers ({N} companies)"

### Legend

Small legend below the chart explaining the visual elements.
