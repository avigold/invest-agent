# PRD 10.3 — Key Ratio Dashboard (Feature Roadmap #5)

## Problem

Financial ratios are buried in plain lists under the "Fundamentals Score" section. Bloomberg and YCharts show P/E, P/B, ROE, and margins in a prominent scannable card grid at the top of the stock page. Missing P/E and P/B entirely.

## Solution

Add a key ratio dashboard card grid between the chart and the scoring sections on StockDetail. Compute P/E and P/B from existing market_cap and financial data. Pull remaining ratios from ML feature_values.

## API Change

Extend `GET /v1/predictions/score/{ticker}` response to include:
```json
{
  "market_cap_usd": 3200000000000,
  "key_ratios": {
    "pe_ratio": 32.5,
    "pb_ratio": 48.2,
    "dividend_yield": 0.005,
    "roe": 0.15,
    "net_margin": 0.25,
    "gross_margin": 0.46,
    "operating_margin": 0.30,
    "revenue_growth": 0.08,
    "eps_growth": 0.12,
    "debt_equity": 1.87,
    "current_ratio": 1.04,
    "fcf_yield": 0.04
  }
}
```

## UI

Card grid: 3 columns desktop, 2 mobile. Categories:
- **Valuation**: P/E, P/B, Market Cap
- **Profitability**: ROE, Net Margin, Gross Margin, Operating Margin
- **Growth**: Revenue Growth, EPS Growth
- **Risk**: Debt/Equity, Current Ratio, FCF Yield

Each card: label, formatted value, colour indicator (green/red/neutral for ratios). Missing values show "—".

## Files Changed

| File | Change |
|------|--------|
| `app/api/routes_predictions.py` | Add `market_cap_usd` + `key_ratios` to score response |
| `web/src/components/KeyRatios.tsx` | New card grid component |
| `web/src/pages/StockDetail.tsx` | Render KeyRatios between chart and scores |

## Acceptance Criteria

1. Card grid visible on `/stocks/AAPL` with P/E, P/B, ROE, margins
2. Values formatted correctly (%, x, $B)
3. Negative values in red, positive profitability in green
4. Responsive layout (3 cols → 2 cols)
5. Stocks without ML data fall back to deterministic component_data
6. `npm run build` succeeds
