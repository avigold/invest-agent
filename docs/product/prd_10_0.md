# PRD 10.0 — Watchlist

## Problem Statement

Users have no way to track a personal set of stocks within Invest Agent. To monitor companies of interest, they must manually navigate to each stock detail page. Bloomberg Terminal and YCharts both provide watchlists as a core feature. A watchlist is the most fundamental personalisation feature for any investment research platform.

## Solution

Add a single default watchlist per user (MVP — multiple named watchlists can come later). Users can add/remove tickers from any stock detail page via a star/bookmark button. A dedicated `/watchlist` page displays all watched stocks with live prices, daily changes, country, sector, and scores.

## Data Model

**New table: `watchlist_items`**

| Column | Type | Constraints |
|--------|------|-------------|
| `id` | UUID | PK, default uuid4 |
| `user_id` | UUID | FK → users.id, CASCADE, NOT NULL |
| `company_id` | UUID | FK → companies.id, CASCADE, NOT NULL |
| `ticker` | String(20) | NOT NULL (denormalised for display) |
| `position` | Integer | NOT NULL, default 0 |
| `added_at` | DateTime(tz) | NOT NULL, default utcnow |

- Unique constraint: `(user_id, company_id)`
- Index: `user_id`
- Max 200 items per user (enforced at API level)

## API Surface

**New file**: `app/api/routes_watchlist.py` — prefix `/v1/watchlist`

| Method | Path | Body | Response | Notes |
|--------|------|------|----------|-------|
| GET | `/v1/watchlist` | — | `[{id, ticker, name, country_iso2, gics_code, position, latest_price, change_1d, change_1d_pct, currency, overall_score, composite_score, ...}]` | Enriched with prices + scores |
| POST | `/v1/watchlist` | `{"ticker": "AAPL"}` | `{id, ticker, position}` | 404 if not found, 409 if duplicate |
| DELETE | `/v1/watchlist/{ticker}` | — | 204 | — |
| GET | `/v1/watchlist/check/{ticker}` | — | `{"in_watchlist": bool}` | For star button state |
| PUT | `/v1/watchlist/reorder` | `{"order": ["uuid1",...]}` | `{"status": "ok"}` | Full reorder |

All endpoints require authentication via `Depends(get_current_user)`.

## Acceptance Criteria

1. `alembic upgrade head` succeeds

6. User can reorder watchlist items
7. Watchlist is per-user (isolated between users)
8. Max 200 items enforced
9. `pytest -q` passes
10. `npm run build` succeeds
